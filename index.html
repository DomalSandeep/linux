Practical 5: Installing and Configure of FTP server

Step 1: Go to the root user and then navigate to the Packages directory

su - root
        cd /media/RHEL….. 1/Packages/
Switches to the root user and navigates to the Packages directory.

Step 2: Check if the vsftpd and the ftp packages are installed. If not, then install the vsftpd and ftp package

rpm -qa | grep vsftpd
        rpm -ivh vsftpd*
        rpm -ivh ftp*
Checks and installs vsftpd and ftp packages if not already installed.

Step 3: Check if vsftpd and ftp is installed once again

rpm -qa | grep vsftpd
        rpm -qa | grep ftp
Verifies if vsftpd and ftp packages are installed.

Step 4: Use the chkconfig command to start the vsftpd services at boot time

chkconfig vsftpd on
        chkconfig –list | grep ftp
Enables vsftpd service to start at boot time.

Step 5: Then navigate to the /var/ftp/pub/ directory using the cd command and create a file for testing ftp.

cd /var/ftp/pub/
        cat -> ftpfile
        (ctrl + d to save and exit)
        To check contents of file
        cat ftpfile
Navigates to the directory and creates a file for testing FTP.

Step 6: Verify IP address of linux machine to be configured as FTP.

ifconfig
Displays the IP address of the Linux machine.

Step 7: Open configuration file and make following changes

vi /etc/vsftpd/vsftpd/conf
Edits the vsftpd configuration file.

Step 8: Then restart the vsftpd service using the service restart command.

service vsftpd restart
        service vsftpd status
        service vsftpd restart
Restarts the vsftpd service to apply configuration changes.

Step 9: Go back to the main directory using the cd command and then login with an anonymous user.

cd
        ftp <ip address>
        Name : anonymous
        Password : (just click enter)
        Then, use ls –a command to view the content of ftp home directory.
        ls -a
        bye (to exit out of ftp)
Logs in as an anonymous user to FTP server.

Step 10: Now allow ftp anonymous write enable by using the setsebool command

getsebool –a | grep ftp
        setsebool –P allow_ftp_annon_write on or = 1
        getsebool –a | grep ftp
Allows anonymous users to write in FTP.

Step 11: Then allow system users to get access to ftp server by turning the ftp_home_dir on using the setsebool command.

getsebool –a | grep ftp
        setsebool –P ftp_home_dir on
        getsebool –a | grep ftp
Allows system users to access the FTP server.

Step 12: By default the /var/ftp is in the ftp root user’s home directory. Check the context of file /var/ftp/pub and change the group and owner to ftp.

ls –ldZ /var/ftp/pub
        chgrp ftp /var/ftp/pub
        chown ftp /var/ftp/pub
        ls –ldZ /var/ftp/pub
Checks and changes the group and owner of the directory.

Step 13: Go to the pub directory using the cd command and create a file named ftptest and enter some text.

cd /var/ftp/pub
        touch T1 T2 T3
        cat > ftptest
        (to save and exit : ctrl +d)
Creates a file named ftptest in the pub directory.

Step 15: Go to the Packages directory and then restart the service of vsftpd and enable it from boot.

Cd /media/RHEL 1/Packages/
        service vsftpd start
        service vsftpd restart
        chkconfig vsftpd on
        chkconfig –list | grep vsftpd
Restarts vsftpd service and enables it to start at boot time.

Step 16: Now, we do the testing as an FTP client

ftp <ip address>
        Name : anonymous
        Password : (just click enter)
Logs in as an anonymous user to FTP server for testing.

Step 17: Then, we disable the anonymous user to use FTP login

cd /var/ftp/pub
        vi /etc/vsftpd/vsftpd.conf
        i) Go to directive anonymous _enable = YES and make it anonymous _enable = NO.
        ii) Go to directive anonymous_upload_enable = YES and make it anonymous_upload_enable = NO.
        (:wq to save and quit)
Disables anonymous user login for FTP.

Step 18: Restart the vsftpd service

service vsftpd restart
Restarts the vsftpd service after making changes.

Step 19: Then, try logging into ftp as an anonymous user.

ftp <ip address>
        Name : anonymous
        Password : (just click enter)
Attempts to log in as an anonymous user to FTP server after disabling anonymous login.

Practical 1: Reverse of a Number

Command 1: Reverse of a number

            echo "Accept number:"
            read num
            rev=0
            rem=0
            if [ $num -le 0 ]; then
                echo "Invalid Number!"
                exit
            fi
            while [ $num -ne 0 ]; do
                rem=`expr $num % 10`
                rev=`expr $rem + $rev \* 10`
                num=`expr $num \/ 10`
            done
            echo "Reverse number is $rev"

Practical 2: Decimal to Binary Conversion

Command 1: Decimal to binary conversion

            echo "Accept number:"
            read deci
            bin=0
            p=1
            rem=0
            while [ $deci -gt 0 ]; do
                rem=`expr $deci % 2`
                bin=`expr $bin + $rem \* $p`
                p=`expr $p \* 10`
                deci=`expr $deci \/ 2`
            done
            echo "Binary number is $bin"

Practical 3: Binary to Decimal Conversion

Command 1: Binary to decimal conversion

            echo "Accept number:"
            read bin
            deci=0
            p=1
            rem=0
            while [ $bin -gt 0 ]; do
                rem=`expr $bin % 10`
                deci=`expr $deci + $rem \* $p`
                p=`expr $p \* 2`
                bin=`expr $bin \/ 10`
            done
            echo "Decimal number is $deci"

Practical 4: Decimal to Octal Conversion

Command 1: Decimal to Octal conversion

            echo "Accept number:"
            read deci
            if [ $deci -lt 1 ]; then
                echo "Invalid number"
                exit
            fi
            oct=""
            rem=0
            while [ $deci -gt 0 ]; do
                rem=`expr $deci % 8`
                oct="$rem$oct"
                deci=`expr $deci \/ 8`
            done
            echo "Octal number is $oct"
